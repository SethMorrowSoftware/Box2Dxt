WHAT IT IS

A real physics engine, wearing xTalk's clothes.

Box2D powers countless games. Box2Dxt packages Box2D v3.1.0 as a drop-in module for OpenXTalk and LiveCode 9.6.3+, so you get true rigid-body simulation — gravity, friction, restitution, joints, sensors — while writing the plain, English-like xTalk you already know.

370+engine handlers
300+friendly kit calls
6paste-and-run examples

  • Works in pixels & degrees — the units you already think in.
  • Bodies ride on ordinary LiveCode controls; the Kit moves them each frame.
  • The native library is bundled in the extension and auto-loads on Windows, macOS & Linux — no download, rename, or sudo.
WHAT YOU GET

Friendly on top. Serious underneath.

Friendly by default

The Kit speaks pixels and degrees, binds bodies to your controls, and runs the loop. b2kQuickStart gives a live, draggable world in one line.

The full Box2D surface

370+ handlers: bodies, shapes, joints, chains, sensors, ray casts, queries, contact events, and world tuning — in metres and radians when you want the metal.

A built-in Game Kit

A player controller (run, double-jump, wall-jump, dash, duck, climb, swim), a scrolling camera, spritesheets, input, and sound — all wired and verified.

Safe by design

Every handle is validated and generation-tagged, so a stale or invalid one is a harmless no-op — getters return zero, actions do nothing. Never a crash.

Cross-platform, built in

The native library for Windows (x64/x86), macOS (universal), and Linux (x86-64/i686) is bundled in the extension and loads automatically — nothing to download, rename, or place by hand.

FAQ

Questions, answered.

Is it free? Can I use it in commercial apps?

Yes. Box2Dxt is MIT-licensed — free for personal and commercial use. Box2D itself is MIT too (© Erin Catto).

Do I need to know C, or compile anything?

No. The native library is bundled inside the extension and loads automatically. Package the extension once, then write plain xTalk — no toolchain, no terminal.

Which platforms does it run on?

Windows (x64/x86), macOS (universal), and Linux (x86-64/i686) — the right library is chosen for you.

OpenXTalk only, or LiveCode too?

Both. It targets OpenXTalk and is compatible with LiveCode 9.6.3+.

Do I need to know physics or maths?

No. The Kit speaks pixels and degrees and runs the simulation loop for you. Spawn a ball, push a body, react to a hit — the maths is handled.

Is it fast enough for real games?

Yes — it's the real, native Box2D engine. The bundled platformer runs seven scrolling levels with a full cast at 60fps.

Can I add physics to controls I already drew?

Absolutely. Point the Kit at any graphic, image, or button and it gains a body, gravity, and collisions — graphics and images even rotate with it.

Where do I get help?

The guides cover everything; the source, issues, and discussions live on GitHub.

get started
GET STARTED

Up and running in four steps.

  1. 1

    Package the extension

    Open box2dxt.lcb in OXT's Extension Builder and click Package — it rolls in the per-platform native libraries for you.

  2. 2

    Install the .lce

    Add the packaged box2dxt.lce in the Extension Manager. The right native library is bundled inside and loads itself — no download, rename, sudo, or /usr/lib.

  3. 3

    Sanity check

    In the Message Box, run put b2Version(). You should see 4 — the engine is wired up.

  4. 4

    Paste an example & reopen the card

    Drop any example script into a stack script and reopen the card. You're simulating real physics.
